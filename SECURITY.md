# Политика безопасности проекта
## Уведомления
Об уязвимостях сообщать через GitHub Issues (метка "security").

## Подход
Применяется принцип DevSecOps:
- Автоматизированные проверки на каждом этапе CI/CD.
- Непрерывное обновление зависимостей.
- Блокировка деплоя при критических проблемах.

## Контроль
Security Summary генерирует общую оценку прохождения проверок.
