# Runbook (восстановление пайплайна)

## Цель
Инструкция по восстановлению и повторному запуску безопасного CI/CD.

## Действия при сбое
1. Проверить вкладку **Actions** → упавший job.
2. Перейти в лог и определить тип ошибки:
   - Ошибка CodeQL → проверить синтаксис кода.
   - Ошибка ESLint → исправить стиль.
   - Ошибка npm audit → обновить уязвимые пакеты.
3. Исправить код и сделать новый `commit`.
4. После успешных проверок — Security Summary обновит процент.
